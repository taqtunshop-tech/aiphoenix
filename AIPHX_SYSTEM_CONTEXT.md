# [AIPHX] SYSTEM CONTEXT & ARCHITECTURE
**PROJECT:** AIPHX (AIPhoenix)
**DOMAIN:** Проектирование, производство и продажа пластинчатых теплообменников (завод «Феникс», г. Ростов).
**STATUS:** Внедрение локальной мультиагентной ИИ-экосистемы.

---

## 1. ЦЕЛЬ СИСТЕМЫ (EXPECTATIONS)
Автоматизация полного цикла обработки входящих заявок (от письма клиента до готового Технико-Коммерческого Предложения в PDF). Ускорение работы с дней до минут. ИИ выступает в роли "Экипажа" корабля, человек-менеджер — "Капитан" (принимает финальные решения).

## 2. ИНФРАСТРУКТУРА И БЕЗОПАСНОСТЬ (FACTS)
*   **Тип развертывания:** СТРОГО Local / On-Premise (Воздушный зазор для коммерческой тайны). Использование Cloud API для расчетов и чертежей КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО.
*   **Оборудование:** Рабочая станция бюджетом ~€20 000 (CPU: AMD Threadripper PRO WRX90, RAM: 256GB+ ECC, GPU: Уровня RTX 6000 Ada / RTX 4090).
*   **Движок инференса:** KTransformers / SGLang / vLLM.
*   **Модели (Гибрид/MoE):** Qwen3 VL (распознавание сканов), GLM-5.2 / Kimi K2.6 / DeepSeek V4.

## 3. АРХИТЕКТУРА РОЯ АГЕНТОВ (AGENTS SWARM)
Система разделена на узкоспециализированных агентов. Каждый выполняет только свою задачу.
1.  `[AIPHX-OPERATOR]` **Оператор-Парсер:** Читает входящие письма и мессенджеры. Распознает сканы опросных листов (Vision). Извлекает температуры, среды, давления, мощность. Ничего не считает!
2.  `[AIPHX-ORCHESTRATOR]` **Оркестратор:** Получает вектор данных от Оператора. Формирует запросы в RAG. Принимает решение, что нужно сделать (например, импортозамещение "Ридан" -> "Феникс").
3.  `[AIPHX-ENGINEER]` **Инженер-Проектировщик:** Главное ядро расчетов. Работает со строгими физическими формулами. Подбирает количество пластин (TK/TL) из каталога, считает площадь поверхности, гидравлическое сопротивление.
4.  `[AIPHX-CLOSER]` **Клоузер (Сейлз):** Получает готовый расчет от Инженера. Обращается в 1С за актуальными ценами на металл и комплектующие. Формирует PDF ТКП и черновик сопроводительного письма.

## 4. БАЗА ЗНАНИЙ (RAG DATABASES)
Векторная база разделена на два контура для исключения галлюцинаций:
*   **STATIC_RAG (Статика):** Неизменные данные. Законы термодинамики, ГОСТы, СНиПы, технические паспорта пластин (предел 200°C, 2.5 МПа), каталоги конкурентов для импортозамещения.
*   **DYNAMIC_RAG (Динамика):** Меняющиеся данные. Синхронизируется с 1С. Текущие цены на сталь/титан, складские остатки (уплотнения, рамы), текущие сроки производства.

## 5. ПРАВИЛА И ОГРАНИЧЕНИЯ ДЛЯ ИИ (STRICT RULES - "БЕЗ ГЛЮКОВ")
1.  **NO HALLUCINATIONS:** Если в `STATIC_RAG` нет нужной пластины под параметры клиента — система ДОЛЖНА ВЫДАТЬ ОШИБКУ и перевести запрос на человека. Запрещено выдумывать характеристики металла или пластин.
2.  **PRICING RULE:** Цены берутся ТОЛЬКО из `DYNAMIC_RAG` (1С). ИИ не имеет права "допускать" или "предполагать" стоимость.
3.  **HUMAN-IN-THE-LOOP:** Система не отправляет ТКП клиенту напрямую. `AIPHX-CLOSER` сохраняет черновик в CRM, статус меняется на "Требует проверки инженером".

## 6. ИНТЕГРАЦИОННЫЕ УЗЛЫ
*   **ERP/Учет:** 1С Предприятие (Цены, остатки, счета).
*   **CRM:** (Автоматическое ведение карточки, смена этапов "Запрос -> Проект -> Расчет -> Оффер").
*   **Коммуникации:** Интеграция с корпоративной телефонией и почтовым сервером (IMAP/SMTP).
